# LC283 — Move Zeroes

## Pattern
Two Pointers — Read & Write Pointer

## Difficulty
Easy

## Confidence
4/5

---

## Problem Statement

Given an integer array `nums`, move all `0`s to the end while maintaining the relative order of the non-zero elements. Do this in-place.

---

## Relationship

LC283 is not a new algorithm.

It is the Read & Write Pointer pattern with

```
Valid Element = nums[i] != 0
```

instead of

```
nums[i] != val
```

---

## Mental Model

Read Pointer scans every element.

Write Pointer stores only non-zero elements.

After the first pass, fill remaining positions from `write` to end with `0`.

---

## Invariant

Everything before Write Pointer is finalized (non-zero, in original relative order).

---

## Algorithm

```
write = 0
for read in range(len(nums)):
    if nums[read] != 0:
        nums[write] = nums[read]
        write += 1
for i in range(write, len(nums)):
    nums[i] = 0
```

---

## Derivation Path

LC27 → change `valid` definition from `!= val` to `!= 0` → LC283.
