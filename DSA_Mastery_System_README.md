# DSA Mastery System — Living Document

> This document is the operational core of the repository. It defines how the system works, why it works, and the non-negotiable rules that make it effective. Read this before every study session.

---

## Objectives

1. **Acquire knowledge** — understand how and why each pattern works.
2. **Retain knowledge** — encode it into long-term memory through spaced repetition.
3. **Retrieve without hints** — solve problems cold, without reading the pattern file first.
4. **Apply under interview conditions** — think aloud, communicate decisions, handle pressure.

---

## The Learning Cycle

Every problem follows this exact sequence. Never skip a step.

```
Learn Pattern
     ↓
Build Mental Model
     ↓
Dry Run (small example, by hand)
     ↓
Write Algorithm (plain English)
     ↓
Code
     ↓
Debug
     ↓
Analyze Complexity
     ↓
Record Personal Mistakes
     ↓
Create Memory Scene
     ↓
Extract Interview Question
     ↓
Same-Day Recall
     ↓
24-Hour Revision
     ↓
3-Day Revision
     ↓
7-Day Revision
     ↓
14-Day Revision
     ↓
Mixed Interview Test
```

## Rules

1. **Pattern first.** Identify the pattern before writing code.
2. **Dry run before coding.** Trace through a small input manually.
3. **Never memorize code.** Build from the algorithm every time.
4. **Record every personal mistake.** A mistake not written down will repeat.
5. **Every session updates this document.** The system is only as current as your last session.
6. **Revision before new problems.** If revisions are due, complete them first.
7. **No hints on revision.** Solve cold — problem statement only.
